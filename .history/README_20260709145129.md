# Heedo Creative Studio

> AI 创意工作室，专为电商广告团队打造。通过可视化节点画布，将产品照片转化为高质量广告图片和视频。

---

## 项目总体介绍

### 产品定位

Heedo Creative Studio 是一个面向电商广告团队的 AI 驱动创意生产平台。团队成员上传产品照片后，通过可视化工作流编排 AI 能力，生成专业级广告图片和视频素材，并支持团队协作与积分管理。

### 技术架构

```
前端（SPA）
├── Vue 3 + TypeScript        # 框架与语言
├── Vue Flow                   # 节点画布
├── Element Plus               # UI 组件库
├── Pinia                      # 状态管理
└── Vite                       # 构建工具

[后端接口预留]
└── REST API / GraphQL        # 未来对接真实 AI 服务和数据库
```

## 页面功能介绍

### 1. 首页 (Home)

入口页面，展示团队概览数据。

| 功能     | 说明                                               |
| -------- | -------------------------------------------------- |
| 统计卡片 | 总作品数、本月生成、消耗积分、团队成员（模拟数据） |
| 快速入口 | 跳转画布/模板/画廊/积分页面                        |
| 最近作品 | 展示最近 6 个创意作品缩略图                        |
| 创作趋势 | 本周柱状图（静态展示）                             |

### 2. 创意画布 (Canvas)

核心页面，可视化工作流编排器。

#### 2.1 节点库

提供 9 种节点，分为 4 个类别：

| 类别 | 节点        | 积分 | 说明                                                 |
| ---- | ----------- | ---- | ---------------------------------------------------- |
| 输入 | 产品图片    | 0    | 上传本地图片作为工作流起点                           |
| 输入 | 产品描述    | 0    | 输入文本提示词                                       |
| AI   | AI 图片生成 | 50   | 模型（快速/标准/高清）+ 输出尺寸                     |
| AI   | AI 视频生成 | 100  | 时长（2/4/6/10秒）+ 运动模式                         |
| 处理 | 风格滤镜    | 10   | 8 种预设：自然/复古/胶片/黑白/暖调/冷调/HDR/赛博朋克 |
| 处理 | 文字添加    | 5    | 文字内容 + 9 宫格位置选择                            |
| 处理 | 智能裁剪    | 5    | 6 种比例：1:1/4:3/16:9/9:16/3:4/自由                 |
| 输出 | 导出图片    | 0    | 下载按钮 + 预览                                      |
| 输出 | 导出视频    | 0    | 下载按钮 + 视频预览                                  |

#### 2.2 数据类型系统

节点连线需类型匹配：

| 类型   | 标识色       | 说明             |
| ------ | ------------ | ---------------- |
| 图片   | `#4ecdc4` 绿 | 图片类数据传递   |
| 文本   | `#a78bfa` 紫 | 提示词等文本传递 |
| 视频   | `#60a5fa` 蓝 | 视频类数据传递   |
| 无类型 | `#9ca3af` 灰 | 无输出节点       |

#### 2.3 智能连线

- 从输出端口拖拽时，自动弹出兼容节点推荐面板
- 点击即可添加节点并建立连接

#### 2.4 工作流执行

- 点击「开始生成」触发完整执行
- 执行前显示积分消耗预估和余额检查
- 拓扑排序确定节点执行顺序
- 每个节点依次显示 idle → running → done 状态
- 整体进度条实时更新

#### 2.5 草稿管理

- 支持多草稿切换、新建、删除
- 保存草稿至当前会话
- 从模板加载时使用 sessionStorage 中转

#### 2.6 协作与分享

- 生成唯一分享链接（`/canvas/share/:token`）
- 公开访问开关
- 成员角色：owner（拥有者）/ consumer（可消耗点数）/ viewer（仅查看）
- 工作流级可消耗积分额度设置

### 3. 创作模板 (Templates)

预设工作流模板库，支持一键加载到画布。

| 类型   | 模板名称     | 流程                                    | 积分 |
| ------ | ------------ | --------------------------------------- | ---- |
| 短视频 | 基础短视频   | 产品图 → 视频生成                       | 100  |
| 短视频 | 场景化短视频 | 产品图 → 图片生成 → 视频生成            | 150  |
| 短视频 | 风格化短视频 | 产品图 → 图片生成 → 风格滤镜 → 视频生成 | 160  |
| 图片   | 电商主图     | 产品图 → 图片生成 → 智能裁剪            | 55   |
| 图片   | 场景化营销图 | 产品图 → 图片生成 → 风格滤镜            | 60   |
| 图片   | 营销海报     | 产品图 → 图片生成 → 文字添加            | 55   |

**模板特性**：

- 场景提示词在卡片阶段即可编辑
- 积分消耗实时预览

## AI 工具使用

### 使用策略

| 原则                    | 说明                               |
| ----------------------- | ---------------------------------- |
| **AI 辅助而非 AI 作主** | AI 生成初稿，人工审核后使用        |
| **复杂逻辑人工设计**    | 架构决策、类型定义由人工完成       |
| **边界清晰**            | 明确标注 AI 生成部分，便于后续维护 |

### 具体使用场景

| 场景         | 使用工具    | 产出                         | 人工修正               |
| ------------ | ----------- | ---------------------------- | ---------------------- |
| 组件代码生成 | Claude Code | Vue 组件结构、模板语法       | 业务逻辑补充、类型对齐 |
| 样式调试     | Cursor      | CSS 样式、颜色变量           | 品牌一致性调整         |
| 算法实现     | Claude Code | 拓扑排序、连接验证           | 边界条件补充           |
| API 文档查询 | WebSearch   | Vue Flow / Element Plus 用法 | 无                     |

### 人工修正记录

| 修正项                | 问题描述                 | 修正内容                                                                      |
| --------------------- | ------------------------ | ----------------------------------------------------------------------------- |
| **连接验证逻辑**      | AI 生成的类型匹配有漏洞  | 重写 `isValidConnectionFn`，补充 sourcePort.type === targetPort.type 严格检查 |
| **拓扑排序算法**      | 边界条件遗漏（孤立节点） | 补充 visited Set 和兜底逻辑                                                   |
| **节点数据响应式**    | 深拷贝导致响应式丢失     | 改用 `nodes.value.map()` 模式                                                 |
| **模板加载冲突**      | 与画布初始化时序冲突     | 改为 sessionStorage 中转 + onMounted 延迟执行                                 |
| **Element Plus 样式** | 组件样式与项目主题冲突   | CSS 变量覆盖 + `:deep()` 选择器                                               |

### 满意度自评

| 模块           | 评分   | 理由                                                 |
| -------------- | ------ | ---------------------------------------------------- |
| 节点类型系统   | ⭐⭐⭐ | 类型安全、扩展性强，新增节点仅需修改 nodeDefinitions |
| 智能连线推荐   | ⭐⭐⭐ | UX 流畅，拖拽→弹窗流程自然                           |
| 工作流执行动画 | ⭐⭐⭐ | 进度反馈清晰，状态转换平滑                           |
| 扇形 Banner    | ⭐⭐⭐ | 纯 CSS 实现，视觉效果惊艳                            |
| 积分扣费流程   | ⭐⭐   | 对话框设计合理，但 mock 逻辑较简单                   |

### 待改进项

| 模块         | 问题                | 改进方向               |
| ------------ | ------------------- | ---------------------- |
| 节点属性面板 | if-else 分支过多    | 策略模式或组件映射表   |
| 图片上传     | Blob URL 跨标签失效 | FileReader 或后端存储  |
| 撤销重做     | 按钮存在但未实现    | Command Pattern 历史栈 |

## 附录：第 7 项课题设计思路

> 本课题未实际开发，设计方案供参考。

### 产品定位

在画布上叠加「效果数据层」，让创作者在编辑工作流时就能看到素材的投放表现，实现**创意 ↔ 效果**闭环。

### 核心场景

1. **选品决策**：哪些产品图生成的广告 CTR 最高？
2. **素材迭代**：A 版本和 B 版本的 ROAS 对比
3. **成本优化**：高 spend + 低 conversion 的素材及时暂停

### 推荐 UI 方案

**MiniMap 热力 + 侧边详情面板组合**：

1. MiniMap 默认显示热力色（绿=好，红=差，灰=无数据）
2. 点击节点后，右侧滑出详细数据面板
3. 支持 7天/30天/自定义时间范围切换
4. 支持同类型节点的 A/B 版本对比

### 数据模型

```typescript
interface AdPerformance {
  creativeId: string; // 素材ID
  impressions: number; // 曝光量
  clicks: number; // 点击量
  ctr: number; // 点击率
  spend: number; // 花费
  conversions: number; // 转化数
  roas: number; // 广告回报率
}
```
