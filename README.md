# Heedo Creative Studio

> AI创意工作室，专为电商广告团队打造。通过可视化节点画布，将产品照片转化为高质量广告图片和视频。

## 项目概述

### 产品愿景

Heedo Creative Studio 是一个 AI 驱动的创意生产平台，帮助电商广告团队：
- 上传产品照片作为创作起点
- 通过可视化工作流编排 AI 能力
- 生成专业级广告图片和视频素材
- 团队协作与积分管理
- 最终从广告表现数据中学习优化

### 核心模块

| 模块 | 说明 | 实现程度 |
|------|------|----------|
| **Canvas (创意画布)** | 可视化节点编辑器，核心工作流编排 | 深度实现 |
| **Gallery (作品画廊)** | 团队作品展示，含扇形 Banner 和瀑布流 | 完整实现 |
| **Templates (创作模板)** | 预设工作流模板，含提示词编辑 | 完整实现 |
| **Credits (积分管理)** | 积分余额、消耗明细、充值套餐 | 完整实现 |
| **Team (团队管理)** | 成员角色权限管理 | 完整实现 |
| **Permissions (权限管理)** | 工作流级协作者与额度控制 | 完整实现 |

## 主动删减功能及取舍理由

### 1. AI 生成能力（模拟代替真实 API）

**删除**：真实的 Midjourney / Stable Diffusion / Sora API 集成

**理由**：
- API 接入需要付费账号、额度申请、网络环境等外部条件
- 核心目标是验证工作流编排逻辑和用户体验，而非 AI 能力本身
- 使用 picsum.photos 模拟图片 + 本地 Blob 模拟视频，足以展示完整的输入→处理→输出流程

**影响**：用户无法获得真实 AI 生成结果，但可完整体验工作流搭建和执行过程

### 2. 数据持久化（内存代替数据库）

**删除**：后端数据库和用户认证系统

**理由**：
- 24小时开发周期内，后端开发会严重拖累核心功能迭代
- 使用 Pinia 状态管理 + sessionStorage 可完成所有页面交互
- 积分消耗、模板加载、草稿保存等功能均已通过前端实现验证

**影响**：刷新页面后数据重置，但不影响功能逻辑验证

### 3. 撤销/重做（按钮但无功能）

**删除**：完整的 Undo/Redo 历史栈

**理由**：
- 工具栏保留了撤销重做按钮位置，保持 UI 完整性
- 实现需要复杂的 Command Pattern 或 Memento Pattern，耗时较长
- 节点删除已有确认提示，降低误操作风险

**影响**：用户需手动重新构建节点

### 4. 第 7 项课题：投放数据联动（未实现）

**删除**：广告投放后的性能数据（impressions/CTR/ROAS）在画布上的展示

**理由**：作为最后一个课题，在时间约束下优先确保前 6 项课题的质量和完整性

**设计思路详见下方专门章节**

## 模糊需求假设说明

### 1. 「产品愿景」的「从广告表现中学习」

**假设**：学习机制体现为积分消耗与创意质量的关联分析

- 高积分消耗的 AI 节点（图片生成 50 分，视频生成 100 分）代表"高投入创意"
- 用户通过观察"高投入 → 画廊高展示 → 实际效果"的循环，自行总结优质创意规律
- 系统层面暂无自动化的 A/B 测试或 ROAS 分析

### 2. 「团队」角色定义

**假设**：采用扁平化的三层权限模型

| 角色 | 可上传图片 | 可运行工作流 | 可充值积分 | 可管理成员 |
|------|------------|--------------|-----------|-----------|
| Admin | ✅ | ✅ | ✅ | ✅ |
| Editor | ✅ | ✅ | ❌ | ❌ |
| Viewer | ❌ | ❌ | ❌ | ❌ |

Canvas 内又细分为 owner/consumer/viewer，与 Team 的 admin/editor/viewer 平行管理。

### 3. 「工作流」执行语义

**假设**：顺序执行 + 拓扑排序

- 节点按依赖关系依次执行（非并行）
- 每个节点有独立的 idle/running/done 状态
- 节点的输出数据（图片URL/文本）自动传递给下游节点
- 导出节点执行后触发下载

### 4. 「模板」的提示词编辑时机

**假设**：模板卡片阶段即可编辑，而非加载到画布后

- 用户在选择模板时即可修改"场景提示词"
- 模板加载时，提示词自动填入"产品描述"节点
- 这样设计让用户在"决策阶段"就参与创意定制

### 5. 「积分」消耗时机

**假设**：运行工作流前预扣费，而非按节点逐步扣费

- 用户点击"开始生成"后弹出积分确认框
- 确认后一次性扣除整个工作流的积分总和
- 全部节点执行成功后正式扣费，取消则不扣

### 6. 「分享链接」的访问权限

**假设**：链接持有者即获得查看权限

- 生成链接后，任何人访问 `/canvas/share/:token` 即可查看工作流
- 不可编辑（Canvas 内 showCollabPanel 逻辑基于 role，但无真实 auth）
- 适合团队内部分享工作流设计思路

## AI 工具使用详情

### 工具优势

| 场景 | 使用工具 | 优势体现 |
|------|----------|----------|
| 组件代码生成 | Claude Code | 快速输出完整的 Vue 组件结构，减少样板代码 |
| 样式调试 | Cursor | 即时预览 + 内联编辑，效率高于浏览器 DevTools |
| API 文档查询 | WebSearch | 检索 Vue Flow、Element Plus 等库的用法 |
| 布局方案设计 | 人工设计 | 确保整体视觉一致性和交互逻辑连贯性 |

### 手动修正内容

1. **Vue Flow 连接验证逻辑**
   - AI 生成的连接验证函数存在类型匹配 bug
   - 手动重写了 `isValidConnectionFn` 和 `validateConnection` 函数
   - 确保 sourcePort.type === targetPort.type 的严格检查

2. **拓扑排序执行算法**
   - AI 生成的拓扑排序有边界条件遗漏
   - 手动补充了 `visited` Set 和孤立节点处理逻辑

3. **节点数据流传递**
   - AI 生成的节点数据更新存在引用问题（响应式丢失）
   - 改用 `nodes.value.map()` 深拷贝模式确保响应性

4. **模板加载机制**
   - 初始模板加载逻辑与画布初始化冲突
   - 改为 sessionStorage 中转 + `onMounted` 延迟执行的方案

5. **样式细节调整**
   - Element Plus 组件与自定义 CSS 的冲突（如 Select 下拉框背景）
   - 大量 CSS 变量覆盖和 `:deep()` 选择器使用

### 满意实现

| 模块 | 满意度 | 理由 |
|------|--------|------|
| 节点类型系统 | ⭐⭐⭐ | 类型安全、扩展性强、新增节点只需在 nodeDefinitions 添加定义 |
| 智能连线推荐面板 | ⭐⭐⭐ | UX 流畅，从端口拖拽到弹出面板的流程自然 |
| 工作流执行动画 | ⭐⭐⭐ | 进度条 + 节点状态 + 整体遮罩的组合反馈清晰 |
| 扇形 Banner 动效 | ⭐⭐⭐ | CSS transform + animation 的纯前端方案，视觉惊艳 |
| 积分扣费流程 | ⭐⭐ | 确认对话框设计合理，但 mock 积分逻辑较简单 |

### 存疑实现

| 模块 | 存疑点 | 改进方向 |
|------|--------|----------|
| 节点属性面板 | 大量 `if-else` 判断节点类型，扩展性差 | 改为策略模式或组件映射表 |
| 图片上传 | 使用 `URL.createObjectURL` 生成 blob URL，跨标签页失效 | 改用 FileReader 或后端存储 |
| 草稿管理 | 仅保存到当前组件内存，刷新丢失 | 接入 IndexedDB 或后端 |
| 撤销重做 | 按钮存在但功能未实现 | 补充 Command Pattern 历史栈 |
| 分享链接 | token 格式随机生成，无法验证真伪 | 接入后端生成带签名的 token |

## 第 7 项课题设计思路（投放数据联动）

> 本课题未实际开发，以下为完整的设计文档。

### 产品定位

在画布上叠加「效果数据层」，让创作者在编辑工作流时就能看到素材的投放表现，实现**创意 ↔ 效果**的闭环。

### 核心场景

1. **选品决策**：哪些产品图生成的广告 CTR 最高？
2. **素材迭代**：A 版本和 B 版本的 ROAS 对比
3. **成本优化**：高 spend + 低 conversion 的素材及时暂停

### 数据模型

```typescript
interface AdPerformance {
  creativeId: string        // 素材ID（对应工作流导出的图片/视频）
  impressions: number       // 曝光量
  clicks: number           // 点击量
  ctr: number              // 点击率 = clicks / impressions
  spend: number            // 花费金额
  conversions: number       // 转化数
  roas: number             // 广告支出回报率 = revenue / spend
  cpc: number              // 单次点击成本 = spend / clicks
}

interface WorkflowDataLink {
  workflowId: string
  creativeId: string
  platform: 'meta' | 'tiktok' | 'google'
  adSetId: string
  campaignId: string
  performance: AdPerformance
  updatedAt: string
}
```

### UI 设计方案

#### 方案 A：节点叠加徽章（推荐）

在画布节点右上角叠加半透明数据徽章：

```
┌─────────────────┐
│  产品图片       │  ← 原始节点
│  ┌───────────┐  │
│  │ CTR: 2.3% │  │  ← 数据徽章（绿色=好，红色=差）
│  │ ROAS: 4.2 │  │
│  └───────────┘  │
└─────────────────┘
```

**优点**：与画布融为一体，不打断工作流编辑流程
**缺点**：空间有限，数据量多时拥挤

#### 方案 B：侧边详情面板

点击节点或选中工作流后，右侧滑出详细数据面板：

```
┌──────────────────────────────────────────┐
│  素材：夏日饮料海报_v2.png               │
│                                          │
│  📊 投放效果                             │
│  ┌────────────────────────────────────┐ │
│  │ 曝光    12.5万    ↑ 23%             │ │
│  │ 点击    2,875     ↑ 18%             │ │
│  │ CTR     2.3%      ↓ 0.2%           │ │
│  │ 花费    ¥458       ↑ 12%            │ │
│  │ 转化    89         ↑ 31%            │ │
│  │ ROAS    4.2        ↑ 0.6            │ │
│  └────────────────────────────────────┘ │
│                                          │
│  📈 趋势图（7天折线）                   │
│  [=======折线图占位=======]             │
│                                          │
│  🔗 关联广告                            │
│  Meta 推广活动 A  │  TikTok 推广活动 B  │
└──────────────────────────────────────────┘
```

**优点**：数据展示完整，支持趋势图和对比
**缺点**：需要切换面板，流程略有中断

#### 方案 C：迷你地图热力图

在画布右下角的 MiniMap 上叠加热力颜色：

- 绿色节点：该素材效果好（ROAS > 目标值）
- 红色节点：该素材效果差（ROAS < 阈值）
- 灰色节点：暂无数据

**优点**：全局视角，一眼识别问题素材
**缺点**：需要 Legend 引导，且色盲用户不友好

### 推荐方案

**采用方案 B（侧边详情面板）+ 方案 C（MiniMap 热力）的组合**：

1. MiniMap 默认显示热力色，帮助用户快速定位问题节点
2. 点击节点后，右侧滑出详细数据面板
3. 支持 A/B 版本对比（同类型节点的多个版本并列展示）
4. 支持时间范围切换（7天/30天/自定义）

### 交互流程

```
1. 用户在 Gallery 选择「查看投放数据」
         ↓
2. 关联工作流，自动跳转 Canvas 并高亮对应节点
         ↓
3. MiniMap 显示热力色（绿色=好，红色=差）
         ↓
4. 用户点击节点，右侧弹出数据面板
         ↓
5. 面板内展示：核心指标 → 趋势图 → 关联广告链接
         ↓
6. 用户可点击「基于此素材创建变体」，复制节点并创建新工作流
```

### 技术实现要点

1. **数据来源**：假设对接 Meta/TikTok/Google Ads API（或模拟数据）
2. **数据更新**：WebSocket 实时推送 或 定时轮询（5分钟间隔）
3. **阈值配置**：ROAS 目标值可自定义（默认 3.0）
4. **A/B 测试**：同节点 ID 的多个版本（v1/v2/v3）自动聚合对比

### 数据联动价值

| 用户角色 | 价值点 |
|----------|--------|
| 设计师 | 「原来赛博朋克风格的 CTR 比自然风格高 40%」 |
| 投放师 | 「这个素材 ROAS 持续下滑，应该暂停并新建变体」 |
| 运营 | 「限时促销banner的 CPM 成本比普通海报高 2 倍」 |
| 团队 | 「跨广告平台对比素材效果，选最优模板」 |
